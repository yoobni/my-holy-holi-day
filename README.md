# 휴무일 캘린더 (로컬 JSON 파일 저장)

내부용 단일 페이지 앱입니다. 데이터는 DB 대신 `data/` 폴더의 JSON 파일에 저장됩니다.

## 실행 방법

```bash
npm install
npm run dev
```

브라우저에서 `http://localhost:3000` 접속.

## 데이터 파일 구조

- `data/users.json`
  - 허용 사용자 목록. 예시 생년월일(`1990-01-01`)은 실제 값으로 교체하세요.
- `data/offdays.json`
  - 휴무일 목록. `date(YYYY-MM-DD) + name + createdAt` 저장.

## 회원정보 수정 방법

`data/users.json`의 `birth` 값을 실제 생년월일(YYYY-MM-DD)로 교체합니다.

## 파일 DB 방식의 한계 (중요)

이 프로젝트는 **로컬 디스크가 유지되는 환경**을 전제로 합니다.
- 서버가 재시작/재배포되면 파일이 초기화될 수 있는 환경(서버리스, 임시 파일 시스템 등)에서는 부적합합니다.

## 보안 관련 주의

내부용이므로 **보안(서명, JWT, HTTPOnly, CSRF 등)을 전혀 고려하지 않습니다.**

## 사용 시나리오

1) 최초 접속 → 회원정보 입력(로컬 저장)
2) 달력에서 휴무일 선택 → 휴무일 지정
3) 달력/목록에서 휴무일 표시 확인
4) 여러 날짜 선택 → 일괄 삭제
5) 로그아웃(로컬 저장 삭제)
